NMatrix

Apr 11th, 2012

We are pleased to announce the first alpha release (0.0.1) of NMatrix, a component of SciRuby.

NMatrix is an experimental linear algebra library for Ruby, written mostly in C. It can be used with or without SciRuby, but is part of the SciRuby project.

NMatrix was inspired by and based heavily upon NArray, by Masahiro Tanaka.

Features

In addition to dense matrices, NMatrix supports two types of sparse storage (list-of-list and Yale).

We support the following data classes:

  • Byte (unsigned 8-bit integer)
  • Integer (8, 16, 32, and 64-bit)
  • Floating point (32 and 64-bit)
  • Complex (64 and 128-bit)
  • Rational (32, 64, and 128-bit)
  • Ruby objects

NMatrix makes use of and exposes the CBLAS API included in ATLAS. However, we have also written BLAS templates for NMatrix-supported data types that are not included in ATLAS or LAPACK (e.g., rationals).

Installation

gem install nmatrix

For complete installation instructions, please see the SciRuby docs.

Developers

The repositories in the SciRuby account on Github contain only stable code which is ready for release. Since we’re in the midst of a major refactoring, you’re better off forking either Chris Wailes’ repository or John Woods’.

Here’s how you would clone the stable version:

git clone git://github.com/SciRuby/nmatrix.git

And this is how you would get John Woods’ version:

git clone git://github.com/mohawkjohn/nmatrix.git

You may also want to play around with the NMatrix fork of rb-gsl:

git clone git://github.com/SciRuby/rb-gsl.git

Bugs

Visit the NMatrix issue tracker.
